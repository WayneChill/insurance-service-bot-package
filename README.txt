================================================================
保險發展小幫手 LINE Bot
合併版：產險助手 + 保服助手
================================================================

【檔案結構】
  app.py          ── 主程式（Flask + LINE Webhook + 指令處理）
  sheets.py       ── Google Sheets 存取層（保服/業務/增員）
  excel_reader.py ── Google Drive Excel 讀取（42003/42004）
  flex_message.py ── 所有 Flex Message 組裝
  scheduler.py    ── APScheduler 每日早報 + 產險推播
  requirements.txt
  Procfile
  Dockerfile
  config.txt      ── 本機設定（Railway 用環境變數）
  license.txt     ── 授權金鑰

================================================================
【Google Sheets 結構】（同一份 Spreadsheet）
================================================================

工作表 1（第一個 tab）── 產險聯絡狀態
  欄位：保單號碼 | 姓名 | 狀態 | 更新時間

工作表「保服案件」── 自動建立
  欄位：案件ID | 客戶姓名 | 服務項目 | 保單號碼 | 狀態 | 備註 | 建立時間 | 更新時間
  狀態值：待處理 / 已聯絡 / 已送出 / 核對中 / 已完成

工作表「信用卡」── 自動建立
  欄位：姓名 | 銀行名 | 卡號前4碼 | 效期 | 備註保單

工作表「業務追蹤」── 自動建立
  欄位：ID | 姓名 | 電話 | 階段 | 備註 | 建立時間 | 更新時間
  階段值：已聯繫 / 建議書 / 約簽約 / 送保單

工作表「增員追蹤」── 自動建立
  欄位：ID | 姓名 | 電話 | 階段 | 備註 | 建立時間 | 更新時間
  階段值：已聯繫 / 約聊聊 / 約簽約

================================================================
【Google Drive 結構】
================================================================

資料夾名稱：保服資料（或直接設定 DRIVE_FOLDER_ID）
  42003.xlsx ── 壽險保單（查詢客戶 + 壽星/周年早報）
  42004.xlsx ── 產險保單（查詢客戶 + 每日到期推播）

================================================================
【Railway 環境變數】
================================================================

必填：
  LINE_CHANNEL_ACCESS_TOKEN   LINE Bot Channel Access Token
  LINE_CHANNEL_SECRET         LINE Bot Channel Secret
  LINE_USER_ID                你自己的 LINE User ID（接收每日早報）
  GOOGLE_SHEET_ID             Google Sheets 試算表 ID
  LICENSE_KEY                 授權金鑰
  GOOGLE_CREDENTIALS_B64      Google 服務帳號 JSON 的 base64 字串

選填：
  DRIVE_FOLDER_ID             Google Drive 資料夾 ID（留空自動搜尋「保服資料」）

產生 GOOGLE_CREDENTIALS_B64 的方法（Windows PowerShell）：
  [Convert]::ToBase64String([IO.File]::ReadAllBytes("credentials.json"))

產生 GOOGLE_CREDENTIALS_B64 的方法（Mac/Linux）：
  base64 -i credentials.json

================================================================
【LINE Webhook URL 設定】
================================================================

Railway 部署網址（insurance-service-bot 專案）：
  主要：https://你的網址.railway.app/callback
  相容：https://你的網址.railway.app/webhook（相容舊 insurance-bot）

LINE Developers Console → Messaging API → Webhook URL：
  填入 /callback 路徑

================================================================
【指令說明】
================================================================

查詢 <姓名>            查看客戶資料和保單
進度 <姓名>            查看保服案件進度
待辦                   顯示所有待處理保服案件
說明                   顯示指令說明

產險                   產險說明（每日早報自動推播）
壽險                   今日壽星/保單周年統計

業務                   業務追蹤列表
增員                   增員追蹤列表

新增業務 姓名 電話 階段   新增業務追蹤
                        階段：已聯繫 / 建議書 / 約簽約 / 送保單
新增增員 姓名 電話 階段   新增增員追蹤
                        階段：已聯繫 / 約聊聊 / 約簽約
更新業務 B001 建議書     更新業務追蹤階段（ID 為 B001 格式）
更新增員 R001 約聊聊     更新增員追蹤階段（ID 為 R001 格式）

新增卡片 姓名 銀行 卡號前4碼 效期 [保單號碼]
                        新增信用卡（保單號碼可省略）
刪除卡片 姓名 銀行 卡號前4碼
                        刪除信用卡

================================================================
【每日早報格式】（每日 08:00 台灣時間自動推播）
================================================================

主人早安！MM/DD 今日待辦如下：

🚨 產險區
▪️ 急件：_ 組（60天內，含已報價/尚未聯絡）
▪️ 追蹤：_ 組
▪️ 新件：_ 組
▪️ 延後：_ 組

🎂 壽險區
▪️ 當日壽星：_ 位
▪️ 保單周年：_ 組

📋 保服區
▪️ 已聯絡：_ 件
▪️ 已送出：_ 件
▪️ 核對中：_ 件

💼 業務區
▪️ 已聯繫：_ 組
▪️ 建議書：_ 組
▪️ 約簽約：_ 組
▪️ 送保單：_ 組

👥 增員區
▪️ 已聯繫：_ 組
▪️ 約聊聊：_ 組
▪️ 約簽約：_ 組

早報之後，系統會繼續推播產險到期 Flex 卡片（60天內未完成件）。

================================================================
【部署步驟】
================================================================

1. 將本資料夾所有檔案推送到 GitHub insurance-service-bot 倉庫

2. Railway → insurance-service-bot 專案 → 設定環境變數（見上方）

3. 手動觸發 Railway 重新部署（Deploy 按鈕）

4. LINE Developers Console → Webhook URL 改為 /callback

5. 發送「說明」測試，確認 Bot 正常回應

6. 觀察 Railway logs 確認每日 UTC 00:00 早報正常執行後，
   再停止 insurance-bot 舊專案的 Railway 服務

================================================================
【注意事項】
================================================================

- Python 3.11（line-bot-sdk 與 3.12+ 不相容）
- 產險 Flex 卡片的 Postback 仍走 /callback，無需修改
- 業務/增員 ID 格式：B001（業務）/ R001（增員）
- 保服案件 ID 格式：C001
- 如果 DRIVE_FOLDER_ID 留空，系統會搜尋名稱為「保服資料」的資料夾
  建議直接填 DRIVE_FOLDER_ID 避免找錯資料夾
